# API Contracts & Response Standards

This document establishes the official API interface standards and envelope guidelines for all endpoints in the DevPhoeniX platform.

## 1. Unified Response Envelope (apiResponse)

To simplify parsing on the client-side and standardize success and error handling, every Next.js API route MUST return a standardized JSON envelope generated by the helper functions in `src/lib/api-utils.ts`.

### Success Response Envelope
```json
{
  "success": true,
  "data": { ... }
}
```

### Error Response Envelope
```json
{
  "success": false,
  "error": "Short description of the error",
  "code": "ERROR_CODE_IDENTIFIER",
  "details": { ... }
}
```

---

## 2. API Helper Utility Functions

Always import from `@/lib/api-utils`:

* `apiResponse.success(data, status = 200)`: Returns a successful response.
* `apiResponse.error(message, code, details, status = 500)`: Returns a standard server error.
* `apiResponse.badRequest(message, code)`: Returns a `400 Bad Request` validation error.
* `apiResponse.unauthorized(message)`: Returns a `401 Unauthorized` session error.
* `apiResponse.notFound(message)`: Returns a `404 Not Found` resource error.

## 3. Active Contracts & Endpoints

### `/api/programs`
* **GET**: Returns the list of all normalized programs.
* **POST**: Creates a new program. Validates title, category, price, and auto-generates a URL slug if not provided.
* **PUT**: Updates an existing program by its slug/id.
* **DELETE**: Deletes a program by id.

### `/api/blog`
* **GET**: Returns the list of all blogs, sorting by publication date.
* **POST**: Creates a new blog post.
* **PUT**: Updates an existing blog post.
* **DELETE**: Deletes a blog post.

### `/api/leads`
* **GET**: Returns CRM leads.
* **POST**: Submits a new lead (e.g. from exit-intent popup or contact forms).
* **PUT**: Updates the pipeline status or adds notes to a lead.
* **DELETE**: Deletes a lead by id.

### `/api/mentors`
* **GET**: Returns list of all mentors.
* **POST**: Registers a new mentor.
* **PUT**: Updates mentor details.
* **DELETE**: Deletes a mentor profile.

### `/api/showcase`
* **GET**: Returns list of student alumni projects.
* **POST**: Publishes a new student project.
* **PUT**: Modifies project metadata or links.
* **DELETE**: Removes a project from showcase.

---

## 4. Error Observability Standard

* All runtime mutations in the API routes MUST catch exceptions and log detailed system errors internally.
* Never return bare, un-stringified Supabase exception objects to the client. Map database errors to clean error messages while preserving logs in terminal output.
